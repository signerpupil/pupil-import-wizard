1. Schüler-Stammdaten

Import der LehrerOffice-Schülerexporte (S_*, P_ERZ1_*, P_ERZ2_*). Erzeugt PUPIL-konforme XLSX.

3. Lehrpersonen-Zuweisungen

3-Step-Wizard: ordnet LO-Lehrpersonen den PUPIL-Referenzen zu (positional mapping).

RegelBeschreibungBeispiel
LP-MATCH-EXACTExakter Match per Kürzel zwischen LO und PUPIL.LO MEI ↔ PUPIL MEI ✓ auto
LP-MATCH-FUZZYFuzzy-Match per Nachname+Vorname (Levenshtein).LO „Müller, A." ↔ PUPIL „Mueller, Anna" → Vorschlag
LP-UNMATCHEDNicht zuweisbare Einträge müssen manuell verknüpft oder übersprungen werden.Listet alle offenen Fälle in Schritt 2
LP-MEMORYManuelle Zuweisungen werden im Correction-Memory gespeichert und beim nächsten Import vorgeschlagen.Einmal „MUE → MEI" gemappt → beim nächsten Import automatisch

5. Globale Regeln (Import-übergreifend)

BereichRegel
PRIVACY Datenverarbeitung100 % lokal im Browser (RAM). Keine Übertragung an Dritte. DSG/GDPR-konform.
EXPORT FormatExcel (.xlsx) als priorisierter Default. Datumsspalten werden mit Zellformat @ (Text) erzwungen, um Auto-Konvertierung zu verhindern.
ROWS ZeilennummernAlle Fehler-Zeilen tragen ein +2-Offset (Header=1, erste Datenzeile=2), damit sie 1:1 zu Excel passen.
SKIP Diakritika-WarnungenSystem-weit deaktiviert — León ↔ Leon erzeugt keine Warnung mehr.
MULTI Mehrere DateienMehrere Quelldateien können in Step 1 zusammengeführt werden, Herkunft via _source_file.
MEMORY Correction-MemoryManuelle und Bulk-Auto-Fixes werden persistiert und beim nächsten Import als Vorschlag angeboten. Matching: exakt ODER identifier-basiert.
UI Fehler-GruppierungFehler werden pro Spalte gruppiert, nach Häufigkeit sortiert. Spezialfälle (Sibling, NameChange, Dedup, ParentConsolidation) werden in eigenen Cards angezeigt und aus der allgemeinen Fehlertabelle entfernt.
